# ELOT AI — Marketing Video Pipeline

Renders a ~3½-minute 1280×720 MP4 from the 18 screenshots in
`../screenshots/` and the voiceover script in `../screenshots/SCRIPT.md`.

The final video is committed at `elot-final.mp4`. To re-render after a UI
change or a script edit, run the pipeline below.

## What it does

```
SCRIPT.md          ──┐
                     ├─► TTS engine ──► audio/NN.mp3  ──┐
screenshots/*.png  ──┤                                  ├─► clips/NN.mp4 ──► elot-final.mp4
                     └────────────────► ffmpeg          ┘   (concat + fade)
                          (Ken Burns zoom + mux)
```

Two voice-generation engines, same orchestrator, same output format:

| Engine    | Where it runs              | Quality              | Cost    | Setup                           |
|-----------|----------------------------|----------------------|---------|---------------------------------|
| `say`     | macOS built-in `say` cmd   | Decent (Samantha/Daniel) | Free  | None — works out of the box     |
| `gcp`     | Google Cloud Text-to-Speech | Excellent (Neural2 / Studio / Chirp 3) | ~$0.10 / run | Open billing, enable TTS API   |

## Quick start (free, no cloud)

```bash
# From the repo root
python3 scripts/make-video.py
```

Auto-detects you don't have `GCP_ACCESS_TOKEN`, falls back to macOS `say`.
Output: `artifacts/video/elot-final.mp4`. Takes ~90 seconds.

## Quick start (Cloud TTS)

```bash
# From the repo root
bash scripts/make-video.sh
```

The wrapper:

1. Reads `gcloud config get-value project`.
2. Enables `texttospeech.googleapis.com` on first run (idempotent).
3. Grabs a fresh access token via `gcloud auth print-access-token`.
4. Invokes `scripts/make-video.py` with `TTS_ENGINE=gcp`.

> **Heads-up — billing must be active.** Cloud TTS won't enable on a
> project with a suspended/closed billing account. If `gcloud billing
> accounts list` shows `OPEN: False`, fix it at
> <https://console.cloud.google.com/billing> before re-running.

## Tuning

All env-var-driven so you can iterate without touching code.

| Env var             | Engine | Default        | Notes                                     |
|---------------------|--------|----------------|-------------------------------------------|
| `TTS_ENGINE`        | both   | auto (`gcp` if token else `say`) | force one or the other          |
| `TTS_VOICE`         | gcp    | `en-US-Neural2-D` | try `en-US-Studio-M` for premium, `en-US-Neural2-J` for female |
| `TTS_VOICE`         | say    | `Samantha`        | try `Daniel` (en_GB), `Karen` (en_AU)     |
| `TTS_SPEAKING_RATE` | gcp    | `0.95`            | 1.0 = natural, 0.85 = slow, 1.1 = fast    |
| `TTS_SPEAKING_RATE` | say    | `175`             | words / minute                            |
| `TTS_PITCH`         | gcp    | `-1.0`            | semitones; negative = deeper              |
| `KEN_BURNS`         | both   | `1`               | `0` for static frames                     |
| `RECACHE_TTS`       | both   | `0`               | `1` to bust the audio cache               |

Examples:

```bash
# Premium Google voice (Studio-quality)
TTS_VOICE=en-US-Studio-M bash scripts/make-video.sh

# Free macOS voice — British
TTS_VOICE=Daniel python3 scripts/make-video.py

# Faster, no zoom, force regenerate audio
KEN_BURNS=0 RECACHE_TTS=1 TTS_SPEAKING_RATE=190 python3 scripts/make-video.py
```

## Outputs

| File                                | Committed? | Source                       |
|-------------------------------------|------------|------------------------------|
| `elot-final.mp4`                    | **yes**    | the deliverable              |

## Upgrade path — Vertex AI Veo (cinematic motion)

Cloud TTS gives you the voice. Veo 3 (image-to-video) can replace the
static-frame-plus-Ken-Burns step with real motion — animating each PNG
into an 8-second clip the way a real product video would.

- API: `aiplatform.googleapis.com` (already enabled on your project).
- Cost: ~$0.50–$0.75 per generated second of video; 18 scenes × 8 s ≈
  $70–$110 / render.
- Reliability: probabilistic — sometimes a scene needs a re-roll.
- Wiring: drop a `synthesize_veo()` next to `synthesize_gcp()` in
  `scripts/make-video.py`, gate on `VIDEO_ENGINE=veo`, and replace the
  `render_clip()` Ken-Burns step with the polled Veo output. Roughly
  ~80 lines of code, one extra Vertex AI endpoint call per scene.

If you want this upgrade once billing is healthy, ping and I'll wire it
in. Until then, the macOS `say` path is one command away.

## Why two engines?

The default GCP TTS path is what most teams will want — better voices,
trivially cheap, easy to swap voices/languages without restarting from
scratch. The `say` fallback exists so the entire video pipeline still
works **today** without a cloud account, billing, API keys, or even
internet — useful for hackathons, offline demos, and quick iteration.
Same script, same output, same `elot-final.mp4`.
